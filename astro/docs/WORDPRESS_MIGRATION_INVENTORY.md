# WordPress Migration Inventory

This document tracks the components, pages, and content that must be migrated from the current WordPress site to the new Astro/Sanity website.

## A. What to export from WordPress

Before shutting down or modifying the live site, ensure the following are exported and securely backed up:

*   pages
*   posts
*   categories
*   tags
*   media library
*   Elementor templates if used
*   menus
*   forms
*   SEO metadata
*   redirects from plugins if any
*   XML sitemap
*   users/authors if blog authors matter
*   Google Search Console top pages
*   GA4 landing pages if available
*   backlinks from SEO tool if available

## B. Priority migration order

1.  Homepage
2.  Kajabi Services
3.  Services overview
4.  Top service pages
5.  Case studies
6.  Contact/free audit
7.  Legal pages
8.  Top traffic blog posts
9.  Remaining blog posts

## C. URL inventory columns

When building out the migration tracking spreadsheet (e.g., in Google Sheets or tracking CSV), include the following columns:

*   Old URL
*   New URL
*   Page type
*   Current status
*   Priority
*   Indexed in Google?
*   Has backlinks?
*   Traffic level
*   Keep/rewrite/remove
*   Redirect target
*   Redirect status
*   Notes
