# Cloudflare Turnstile Setup

Turnstile is used to protect forms from spam and bots without presenting an annoying CAPTCHA. It provides an invisible or non-intrusive challenge that verifies real users, improving user experience over traditional CAPTCHAs.

## Setup Instructions

1. **Create Turnstile Site:** Go to your Cloudflare Dashboard > Turnstile. Add the domain.
2. **Keys:** You will receive two keys:
   - **Site Key:** (`PUBLIC_TURNSTILE_SITE_KEY`) - Safe to expose. Put this in `.env` or CF dashboard. It renders the widget.
   - **Secret Key:** (`TURNSTILE_SECRET_KEY`) - **NEVER EXPOSE THIS**. This must only live on the server/function.

## Server-Side Validation Requirement

A frontend widget alone DOES NOT stop spam. A bot can submit the form directly.
- The Turnstile token generated by the widget must be passed to your backend.
- The backend makes a POST request to Cloudflare's `siteverify` API using the `TURNSTILE_SECRET_KEY`.
- If the token is valid, the backend proceeds to save the lead.

## Token Expiry
Turnstile tokens expire after a few minutes. If a user leaves the form open for a long time before submitting, validation will fail. Handle this gracefully in the frontend or backend response.

## Testing
- Without the secret key configured on the backend, you cannot fully test Turnstile.
- You can test failure by submitting an empty token or an invalid one to ensure the backend rejects it.

## Important Reminders
- **Do not expose secret key:** The `TURNSTILE_SECRET_KEY` must never be sent to the frontend.
- **Widget alone is not enough:** Do not claim spam protection is complete with the frontend widget only. Server-side validation is strictly required.
