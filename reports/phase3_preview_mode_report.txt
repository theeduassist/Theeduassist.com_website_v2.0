Phase 3 Preview Mode Report
===========================

- Preview Route Proposal: `/preview/blog/[slug]/` and `/preview/service/[slug]/`
- Security Model: Access to preview pages must be protected by a query string secret (e.g., `?secret=MY_SECURE_TOKEN`) or an authentication cookie. If the secret is missing or invalid, the route returns a 401 or 404.
- Cache Behavior: Preview routes must return headers `Cache-Control: no-store, no-cache, must-revalidate`. They should never be cached on Vercel's Edge or the browser.
- Sanity Client Used: A dedicated Sanity client with `useCdn: false`, `perspective: 'previewDrafts'`, and authenticated with a read token (`process.env.SANITY_READ_TOKEN`). This token must NEVER be exposed to the client.
- Risks:
  - If a preview route is built without `noindex` or `no-store`, a search engine might index draft content.
  - If the Sanity read token leaks into the browser bundle (e.g., prefixed with `PUBLIC_`), it poses a security risk.
  - Since Astro is fully static right now, implementing a server-rendered preview route requires switching Astro to `output: 'hybrid'` and exporting `prerender = false` on preview endpoints.
- Manual Editor Testing Steps:
  1. Editors generate a secure link from Sanity Studio using a Preview plugin.
  2. The link opens `https://www.theeduassist.com/preview/blog/my-draft?secret=TOKEN`.
  3. Verify the browser shows draft content without caching it on reload.
  4. Verify the page source includes `<meta name="robots" content="noindex, nofollow" />`.

Acceptance Check:
- Public visitors cannot access drafts: Required (secret protection).
- Preview cannot be indexed: Required (`noindex` tag).
- Preview does not pollute public cache: Required (`no-store` headers).
