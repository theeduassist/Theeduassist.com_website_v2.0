# Accessibility Report

All trust pages meet baseline accessibility requirements:
- Single H1 present
- Semantic landmark elements (main, header, section, footer)
- Logical heading hierarchy (H2 for main sections, H3 for subsections)
- Native details and summary tags used for FAQs, providing accessible disclosure widgets without JS.
- Adequate contrast ratios maintained with brand colors.
- Responsive design ensuring readability across device sizes (360px to 1920px tested).
- Content avoids communicating information purely via colour.
- Breadcrumbs implemented for easy navigation.
