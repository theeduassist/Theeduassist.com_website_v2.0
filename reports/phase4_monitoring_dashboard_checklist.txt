# Phase 4 - Monitoring Dashboard Checklist

## Daily Monitoring (First Week After Deploy)
The following metrics must be checked daily to ensure post-deployment stability and indexing health:

*   [ ] **Sanity API Requests:** Monitor for unexpected spikes that could indicate runaway client-side fetching or build loops.
*   [ ] **Sanity API CDN Requests:** Ensure usage is within quota limits (expected to be low since `useCdn: false` is used for build-time fetching).
*   [ ] **Vercel Error Rate:** Check the Vercel dashboard for 500 errors, specifically looking for SSR crashes related to missing data fallbacks.
*   [ ] **Vercel Edge Requests:** Monitor traffic volume to gauge if new indexable pages are being hit by bots/users.
*   [ ] **Form Submissions:** Test or monitor the Audit and Contact forms to ensure lead flow is uninterrupted.
*   [ ] **Sitemap Status:** Check `https://www.theeduassist.com/sitemap.xml` to ensure it returns 200 OK and contains the expected URLs.
*   [ ] **Robots Status:** Check `https://www.theeduassist.com/robots.txt` to ensure it returns 200 OK and has correct allow/disallow rules.

## Weekly Monitoring
These metrics should be reviewed weekly to track long-term SEO and conversion goals:

*   [ ] **Google Search Console Indexing:** Monitor the "Indexed" vs. "Not indexed" chart.
*   [ ] **404s:** Review the "Not Found" error report in GSC and Vercel logs to catch new broken links.
*   [ ] **Crawled - currently not indexed:** Monitor this GSC group to see if recent page quality improvements are working.
*   [ ] **Discovered - currently not indexed:** Monitor this GSC group to track crawl budget efficiency.
*   [ ] **Top Landing Pages:** Review GA4 to see which pages are driving initial entry.
*   [ ] **Top Blog Posts:** Track which content roadmap topics are performing best.
*   [ ] **CTA Clicks:** Review GA4 events for `click_get_24_48_review` and other key interactions.
*   [ ] **Core Web Vitals:** Check GSC or PageSpeed Insights to ensure performance remains high.
*   [ ] **Sanity Quota Trend:** Ensure overall API and bandwidth usage remains sustainable.

## Manual Dashboard Setup Needed
The site owner must complete these steps manually in their respective platforms:

1.  **Vercel Web Analytics:** Enable in the Vercel project settings if not already active.
2.  **Vercel Speed Insights:** Enable in the Vercel project settings.
3.  **Google Search Console:** Submit the updated `sitemap.xml` in the Sitemaps report.
4.  **Google Analytics (GA4):** Ensure the Data Stream is active and custom events (listed in Task 4) are configured.
5.  **Cloudflare/Vercel Firewall Review:** Check WAF rules to ensure legitimate bots (Googlebot) are not being blocked while malicious traffic is mitigated.
6.  **Sanity Usage Check:** Log into the Sanity manage dashboard to set up billing/usage alerts.
