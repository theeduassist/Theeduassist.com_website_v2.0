## Implementation Rules & Privacy Guardrails
1.  **Do Not Break Forms:** Event tracking scripts (e.g., `onsubmit` handlers) must not interfere with the default form submission behavior or validation logic.
2.  **No PII in Analytics:** Do not send Personally Identifiable Information (PII) such as names, emails, phone numbers, or free-text message contents to GA4 as event parameters. Track only the *action* (e.g., `form_submitted: true`).
3.  **Privacy Policy Consistency:** Ensure the existing privacy policy accurately reflects the use of Google Analytics for conversion tracking.
4.  **No Sensitive Data:** Never expose sensitive form fields to analytics.

## Manual Setup Steps (if relying on GTM or GA4 Dashboard)
If these events are not hardcoded into the button `onclick` attributes, they must be set up manually in the GA4 dashboard using the "Events" > "Create Event" feature based on existing page views or button click classes.
