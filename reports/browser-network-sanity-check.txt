# Browser Network Sanity Check

This report verifies that public pages no longer make runtime client-side requests to the Sanity API.

## Goal
Normal public pages must show zero browser requests to `sanity.io` or `apicdn.sanity.io` after the page load.

## Test Procedure (To be performed manually after deployment)
1. Open Google Chrome.
2. Open Chrome Developer Tools (F12) and navigate to the **Network** tab.
3. In the Network filter box, type `sanity`.
4. Hard refresh (Ctrl+F5) the following pages on the live production site:
   - Homepage (`/`)
   - Blog Index (`/blog/`)
   - A specific blog post (`/blog/what-is-a-learning-management-system/`)
   - Services (`/services/`)
   - Contact Page (`/contact/`)

## Expected Result
* The Network tab should remain completely empty when filtered by `sanity`.
* No XHR/Fetch requests to `https://*.apicdn.sanity.io/*` should occur.
* All content should be fully rendered in the initial HTML document served by Vercel/Cloudflare.

## Forms Check
1. Navigate to `/contact/` and `/book-free-audit/`.
2. Ensure the forms still render.
3. Test a form submission to verify Turnstile/Firebase logic remains unaffected by the Sanity static optimization.
