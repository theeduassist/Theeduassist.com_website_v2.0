1. Executive Summary
The primary goal of this task was to identify and remediate a critical SEO blocker causing `noindex, nofollow` to appear on global public pages (e.g. `/`, `/kajabi-services/`), audit `X-Robots-Tag` behavior, fix the XML sitemap generation, enforce Sanity API protections, and ensure no security leaks existed in the environment configuration or builds. All items were successfully accomplished.

2. Root Cause of Global Noindex
In `src/layouts/Layout.astro`, the `isMainDomain` flag determines whether `noindex` should be dynamically applied (as a safeguard against indexation of Vercel preview domains). It relied on `import.meta.env.PUBLIC_SITE_URL`. During static site generation (the build phase), `import.meta.env.PUBLIC_SITE_URL` evaluated as `undefined`, causing the `siteUrl` fallback to equal `''`. This resulted in `isMainDomain` being set to `false`, ultimately forcing every page to inherit `shouldNoIndex = true` and `<meta name="robots" content="noindex, nofollow" />`.

3. Files Changed
- `src/layouts/Layout.astro`: Modified `siteUrl` fallback from `''` to `'https://www.theeduassist.com'` and explicitly checked equality, ensuring default behavior matches the production domain during SSG builds.
