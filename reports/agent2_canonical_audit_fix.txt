Canonical Audit & Fix
=====================

Issue: Google Search Console reported a malformed canonical URL on `/pricing/`:
`https://www.theeduassist.com/https:/www.theeduassist.com/pricing/`

Root Cause:
`src/pages/pricing/index.astro` passes the full URL `https://www.theeduassist.com/pricing/` to the `canonicalPath` prop of `<Layout>`. The `Layout.astro` then passes it to `buildCanonicalUrl(canonicalPath)` in `src/lib/seo.ts`. The old `buildCanonicalUrl` assumed all inputs were relative paths. If you gave it `https://www.theeduassist.com/pricing/`, it would ensure it starts with `/` making it `/https://www.theeduassist.com/pricing/`, then append it to `site.url` creating `https://www.theeduassist.com/https:/www.theeduassist.com/pricing/` because it replaced double slashes.

Fix Applied:
Updated `src/lib/seo.ts:buildCanonicalUrl` to detect if the `path` argument starts with `http://` or `https://`. If it does, it extracts the pathname from the URL and then combines it safely with the canonical `site.url`, ensuring that no non-www or duplicated domains are ever used, and it always ends in a trailing slash. If it's relative, it safely prepends the `site.url`. Empty paths fallback to `https://www.theeduassist.com/`.

Acceptance Confirmed:
The fix handles full URL inputs correctly and outputs well-formed canonical tags, maintaining the `www` canonical domain.
