# WordPress Migration Strategy

1. Preserve important old WordPress URLs using 301 redirects only when a relevant new target exists.
2. Do not redirect all old URLs to homepage.
3. Do not create fake pages just to match old URLs.
4. Do not expose old WordPress plugin/template URLs publicly.
5. Do not create public tag pages from messy WordPress tags yet.
6. Do not add old redirected URLs to the new sitemap.
7. Keep sitemap focused on canonical new Astro URLs only.
8. Keep canonical URLs aligned with new final routes.
9. Old blog posts should be migrated into Sanity only after review.
10. Old media should be compressed and renamed before reuse.
11. WordPress templates, MailPoet links, archive sitemap URLs, job taxonomy URLs, and typo tags should be classified, not published.
